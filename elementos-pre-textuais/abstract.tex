The automation of software development and delivery processes has become essential for organizations seeking to reduce delivery cycles without compromising quality, stability, and security. This work proposes and analyzes a reference architecture for implementing CI/CD pipelines in cloud environments, using Jenkins and Kubernetes as the operational foundation, DORA metrics as a performance assessment mechanism, and Zero Trust principles to strengthen security. The methodology combines bibliographic research and experimental study, considering ephemeral agents in Kubernetes, node segregation between On-Demand and Spot instances, static code analysis, vulnerability scanning, artifact signing, and continuous compliance with SPIRE. The results indicate that the proposed architecture contributes to greater scalability, execution isolation, computational cost reduction, and mitigation of threats associated with the software supply chain. The validation mechanisms introduced low operational impact while increasing traceability and trust in deployed artifacts. It is concluded that the integration of DevOps, Kubernetes, DORA metrics, and Zero Trust is a viable approach for building modern continuous delivery platforms.

\textbf{Keywords}: DevOps. CI/CD. Kubernetes. DORA Metrics. Zero Trust. SPIRE.
